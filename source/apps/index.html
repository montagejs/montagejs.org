Made with Montage

The following case study will get you some insights how a Montage app was built and what impact it had.

Scratchpad (Chrome)

Scratchpad 3.0 is the first shipping Montage application. It is a Note taking Chrome application developed for Google that synchronizes with Google Drive. This version added a few features, an updated visual design, removed the Closure framework and was almost completely re- implemented with Montage.

Thanks to Montage’s modular structure and HTML5 Templates, the skeleton of the app was in place in one day, which allowed the 5 person team to work in parallel from the very beginning, including CSS styling. The project was feature complete in 5 dev-months. About 75% of the application was re-implemented with a 25% reduction in code size.

Tips & Tricks (Chrome OS)

The second Montage project done for Google is the Tips & Tricks application which is the first user experience on a Chrome device. This is a hosted application with a custom Loader. It uses Montage Flow component and was coded in 4.5 dev-months, including UI explorations.

Clicking on one topic brings more details in a light box without loosing the linear exploration with cues on the sides. The topic detail navigation is presented using Montage’s Flow component which allows to place any component on a custom 3D bezier path. Thanks to Flow’s frustum culling algorithm, components that are not visible are not rendered, allowing the content to be loaded lazily, improving application load time.

Create Montage Apps

For more infos how to create your own Montage Apps, head over to the Montage Docs. With the Minit (Montage Initializer) tool you can kickstart your application by generating all files and basic structure.
